# Discord 알림 정책

## 링크

- 주간 브리핑, D-1, D-Day, 테스트 발송에는 사업당 사업 대시보드 링크만 하나 넣는다.
- 링크 형식은 `/projects/{projectId}`다.
- 개별 지출 상세(`/expenses/{expenseId}`)와 일정 관리(`/schedules`) 링크는 Discord 메시지에 넣지 않는다.

## 발송

- 월요일 09:00 Asia/Seoul: 담당자 채널의 기업별 주간 브리핑 스레드에 해당 주 일정을 추가한다. 일정이 없는 사업에는 일정 섹션을 표시하지 않는다.
- 매일 09:00 Asia/Seoul: 일정 전날에는 D-1, 일정 당일에는 D-Day 개별 알림을 담당자 채널에 보낸다.
- 수동 테스트는 주간 브리핑과 같은 형식으로 보내며 제목에 `🧪 테스트`를 표시한다.

## 안전 규칙

- 일정·알림 종류별 발송은 한 번만 허용한다.
- 일정 수정·삭제 뒤 정정 메시지를 보내지 않는다.
- `@` 멘션과 Markdown 입력을 안전하게 이스케이프하고 Discord 멘션을 차단한다.
- 외부 요청 결과가 불명확하면 자동 재발송하지 않고 `검토 필요`로 기록한다.
