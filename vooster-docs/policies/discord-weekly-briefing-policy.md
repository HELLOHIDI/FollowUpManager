# Discord Weekly Briefing Policy

## Delivery hierarchy

For each account-manager channel and company, create exactly one parent message and one weekly thread.

```
<company> <month>월 <week>주차 지출 업무 브리핑
└─ company weekly thread
   ├─ project A expense briefing message
   ├─ project B expense briefing message
   └─ project C expense briefing message
```

- A company is the thread boundary. Do not create one thread per project.
- A project is the message boundary inside the company thread. Do not combine two projects in one message.
- Every project message includes its project dashboard link and the expense stage plus detailed status.
- The action-plan table is not part of the weekly briefing.
- An expense without detailed status is displayed as `사전준비`.

## Delivery safety

- Scheduled delivery checkpoints the company parent, its thread, and each project-message chunk.
- Manual tests use the same hierarchy as scheduled delivery and add only the test marker.
