# FuManager 기업 등록 정책

상태: Slice 1 구현 기준  
기준일: 2026-06-22

## 1. 범위

한 개의 인증된 내부 계정은 여러 기업을 등록하고 수정할 수 있다. 기업 등록과 수정은 보호 경로 `/settings/company`에서 수행한다.

이번 Slice에는 기업 삭제, 임시저장, 대표자명, 담당자 이메일, 주소, 업태, 업종, 사업 등록, 기업·사업 사이드바, 운영 대시보드 연결을 포함하지 않는다. 기업 아래 사업 목록과 선택된 사업의 운영 대시보드는 프로젝트 또는 대시보드 Slice에서 구현한다.

## 2. 저장 필드

사용자가 입력하고 저장하는 기업 정보는 아래 일곱 항목뿐이다.

| 필드 | DB 필드 | 정책 |
| --- | --- | --- |
| 담당자 | `account_manager` | 필수, 고정 담당자 목록 중 1명 |
| 기업명 | `company_name` | 필수, 공백 제거 후 1~100자 |
| 회사 형태 | `business_type` | 필수, `corporation` 또는 `sole_proprietor` |
| 기업규모 | `company_size` | 필수, `medium_enterprise`, `small_enterprise`, `micro_business`, `unknown` |
| 사업자등록번호 | `business_registration_number` | 필수, 하이픈 제거 후 숫자 10자리, 전체 기업에서 고유 |
| 설립일 | `founded_at` | 필수, 유효한 날짜이며 서울 기준 오늘 이후 불가 |
| 법인등록번호 | `corporate_registration_number` | 법인만 필수, 하이픈 제거 후 숫자 13자리 |

대표자명과 담당자 이메일을 포함한 제외 필드는 nullable로 남기지 않고 스키마와 API 계약에서 제거한다.

담당자는 권한이나 로그인 계정이 아니라 `/projects` 홈에서 기업을 묶어 보는 내부 관리용 배정값이다. 기업 1개에는 담당자 1명만 배정한다.

## 3. 조건부 검증

- `corporation`은 법인등록번호가 반드시 있어야 한다.
- `sole_proprietor`는 법인등록번호를 저장하지 않으며 DB 값은 `NULL`이다.
- 사업자등록번호는 `000-00-00000` 형식 입력을 허용하되 서버와 DB에는 숫자 10자리로 저장한다.
- 법인등록번호는 `000000-0000000` 형식 입력을 허용하되 서버와 DB에는 숫자 13자리로 저장한다.
- 사업자등록번호 중복은 HTTP 409로 응답한다.
- 검증은 UI, API Zod 경계, DB constraint에서 중복 적용한다.
- 담당자는 아래 고정 목록 중 하나여야 한다: 정현정, 박종열, 류희재, 허진석, 이영준, 주재형, 이정준.

## 4. 상태 정책

`profile_status`는 클라이언트 입력이 아니라 서버가 계산한다.

- `company_size = unknown`: `review_required`
- 그 외 유효한 기업: `complete`

`incomplete`, `invalid`, 임시저장 상태는 사용하지 않는다. 필수 정보가 유효하지 않으면 저장 자체를 거부한다.

## 5. API와 권한

```text
GET   /api/companies
POST  /api/companies
GET   /api/companies/:companyId
PATCH /api/companies/:companyId
```

- 모든 경로는 인증 이후에만 실행한다.
- MVP는 단일 내부 계정 운영을 전제로 하며 인증된 내부 계정에 전체 기업 목록과 상세를 반환한다.
- 생성·수정은 서버의 명시적 필드 allowlist만 DB에 전달한다.
- 기업 소유권·멤버십·다중 계정 격리는 이번 Slice에 없으며, 다중 사용자 지원 전에 별도 권한 모델을 도입해야 한다.
- 삭제 API와 삭제 UI는 이번 Slice에서 제공하지 않는다.

## 6. 화면 정책

- 좌측에는 현재 계정의 등록 기업 목록, 우측에는 선택 기업의 수정 폼을 표시한다.
- 기업이 없으면 빈 상태와 새 기업 등록 폼을 함께 표시한다.
- `기업 추가`는 기존 선택을 해제하고 빈 등록 폼을 유지한다.
- 저장하지 않은 변경이 있을 때 기업 전환 또는 새 등록을 시도하면 폐기 확인을 제공한다.
- 회사 형태가 법인일 때만 법인등록번호 입력을 노출한다.
- 기업규모가 `unknown`이면 저장은 허용하고 `기업규모 확인 필요` 배지를 표시한다.
- `/projects` 홈은 담당자별로 기업을 그룹핑하고, 기업 추가는 `기업 추가하기` 버튼으로 `/settings/company?mode=create`에 진입한다.
- 저장 성공 후 서버 응답을 목록과 상세 캐시에 반영하고 토스트를 표시한다.

## 7. 완료 조건

- 한 계정에서 두 개 이상의 기업을 생성하고 다시 조회할 수 있다.
- 개인사업자와 법인의 조건부 법인등록번호 규칙이 UI·API·DB에서 동일하다.
- 담당자, 기업명, 회사 형태, 기업규모, 사업자등록번호, 설립일, 법인등록번호 외 사용자 입력 필드가 저장 계약에 없다.
- 생성 후 새로고침, 수정 후 새로고침에도 저장 결과가 유지된다.
- 마이그레이션, 스키마 계약, API, UI, 통합 E2E와 전체 품질 검사가 통과한다.
